## Naming convention

Pillars are major iterations. Letter suffixes increment monotonically — each new iteration takes the next letter, regardless of whether it's on a new corpus or a refinement of the same one. Pillar generation number (3, 4, …) is reserved for larger structural shifts (new architecture, new search algorithm, etc.).

- **pillar3a**: sharp_25_epoch_12, trained on V12 corpus with target_temperature=0.5. Confirmed.
- **pillar3b**: next student. Trained on V13 corpus (selfplay+crisis from pillar3a + MCTS).
- **pillar3c, pillar3d, …**: subsequent iterations, each typically on a fresh corpus generated by the previous pillar.

(Historical note: the previous `pillar3a-v3` was a SpatialValueHead experiment, abandoned. Letters reused here for a different concept.)

```bash
# Reference (old head):
python -m alphatrain.scripts.eval_parallel \
    --model alphatrain/data/pillar3a.pt \
    --value-head-path alphatrain/data/value_head_v12_v12targets.pt \
    --simulations 100 --top-k 30 --batch-size 8 \

    --early-stop --seeds $(seq 0 99)

# Candidate (new head):
python -m alphatrain.scripts.eval_parallel \
    --model alphatrain/data/pillar3a.pt \
    --value-head-path alphatrain/data/value_head_pillar3a.pt \
    --simulations 100 --top-k 30 --batch-size 8 \
    --device mps --workers 16 --max-turns 20000 --q-weight 2.0 \
    --early-stop --seeds $(seq 0 99)
```

**Decision:**
- New ≥ +5% over old on mean OR P50 → proceed with new head for V13 generation
- New within ±3% of old → either works; default to new (it's by definition matched to backbone)
- New < old by >5% → retraining failed; debug before V13 commit
